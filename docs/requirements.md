# 요구사항 명세서 (Requirements Specification)

본 문서는 기능 요구사항(FR), 비기능 요구사항(NFR), 보안 요구사항(SR)을 정의한다. 각 요구사항은 구현·테스트·보고서에서 추적 가능하도록 ID로 관리한다.

## 3. 보안 요구사항 (Security Requirements)

### 인증/세션

| ID | 요구사항 |
|----|---------|
| SR-01 | 비밀번호는 bcrypt/Argon2로 해시 저장 |
| SR-02 | 로그인 실패 횟수 제한 |
| SR-03 | 세션/리프레시 쿠키에 HttpOnly·Secure·SameSite 적용 |
| SR-04 | 로그아웃 시 토큰 무효화 |
| SR-05 | 관리자 계정 권한 분리 |

### 접근제어

| ID | 요구사항 |
|----|---------|
| SR-06 | 상품 수정/삭제는 작성자만 |
| SR-07 | 채팅방 조회는 참여자만 |
| SR-08 | 거래 상태 변경은 당사자만 |
| SR-09 | 관리자 API는 관리자만 |
| SR-10 | 모든 객체 ID 접근 시 소유자 검증 |

> **OWASP Top 10 — Broken Access Control**: 상품 ID·채팅방 ID·거래 ID를 직접 다루는 본 서비스의 핵심 취약점. 서버 측에서 반드시 소유자/참여자 검증.

### 입력 검증/인젝션

| ID | 요구사항 |
|----|---------|
| SR-11 | 모든 입력값 서버 검증 |

| SR-15 | 클라이언트의 role·price·userId 신뢰 금지 |

### 파일 업로드

### 결제/안전거래

|----|---------|

### API 보안

| ID | 요구사항 |
|----|---------|
| SR-34 | 모든 중요 API 인증 확인 |
| SR-35 | 객체 ID 기반 API 소유자 검증 (BOLA) |
| SR-36 | 관리자 API 별도 권한 검사 |
| SR-37 | 결제·로그인·채팅 API Rate Limit |

| SR-39 | 응답에서 불필요한 사용자 정보 제거 |

> **OWASP API Security Top 10 — BOLA**: 객체 ID 조작으로 타인 데이터 접근. 상품·채팅방·거래·결제 API 전부 소유자 검증 대상.
> **API4 (자원 소비 제한 없음)**: 결제·인증·알림 API는 Rate Limit 필수 — CPU/메모리/외부 비용 과소비 방지.
